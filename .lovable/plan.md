
## מה נבנה — עורך פריסות מקצועי

מערכת שמאפשרת לך לבנות פריסות משלך לחדר שיתוף המסך בצורה ויזואלית (drag/resize של פאנלים על גבי קנבס), לשמור אותן בענן, ולהחליף ביניהן מתוך המתג הקיים ליד שאר הפריסות המובנות.

## חוויית משתמש

1. **מצב "עריכת פריסות"** — כפתור חדש (עיפרון) ליד מתג הפריסות פותח מצב עריכה מעל הפריסה הנוכחית.
2. **קנבס עם רשת (12 עמודות × 8 שורות)** בפריסת מסך מלא. כל אלמנט הוא בלוק שאפשר:
   - לגרור (drag)
   - לשנות גודל מכל פינה (resize handles)
   - להצמיד לרשת (snap to grid)
   - לשכפל / למחוק / לשלוח לרקע / להביא לחזית (z-index)
3. **פלטת רכיבים** בצד — גרור-שחרר כדי להוסיף:
   - קנבס שיתוף המסך (חובה — אחד לפריסה)
   - רשימת משתתפים (כרטיסים / רשת אווטרים / רצועה קומפקטית)
   - סרגל כלי ציור (אופקי / אנכי)
   - רצועת ריאקציות אמוג'י
   - שעון + אינדיקטור LIVE
   - קוד חדר
   - HUD איכות/השהיה
   - כותרת (טקסט חופשי)
   - כפתור פעולה (הזמן / הפסק שיתוף / מסך מלא)
4. **פאנל מאפיינים** לכל אלמנט נבחר: ערכת נושא (בהיר/כהה/זכוכית), עיגול פינות, שקיפות, גבול, צל, יישור פנימי.
5. **הגדרות פריסה גלובליות**: רקע (מלא / גרדיאנט / רשת), ערכת נושא, ריווח בין רכיבים.
6. **תצוגה מקדימה חיה** — האלמנטים מציגים תוכן אמיתי (השעון רץ, האמוג'י פועל) גם במצב עריכה.
7. **סרגל עליון**: שם הפריסה · שמור · שמירה כחדש · בטל/חזור (undo/redo) · יציאה.

## החלפה בין פריסות

מתג הפריסות בפינה יציג:
- 3 הפריסות המובנות (סטודיו / מרכז פיקוד / קנבס צף)
- קו הפרדה + כל הפריסות שיצרת (עם תמונה קטנה של הפריסה + שם)
- כפתור "+ פריסה חדשה" שפותח את העורך

## שמירה בענן (Lovable Cloud)

- טבלה `custom_layouts`: `id`, `user_id`, `name`, `spec` (JSON של הבלוקים), `thumbnail` (base64 קטן), `created_at`, `updated_at`.
- RLS: כל משתמש רואה/עורך רק את הפריסות שלו.
- דורש התחברות — נוסיף מסך התחברות פשוט (אימייל+סיסמה + Google) אם עוד לא קיים.

## פרטים טכניים

- ספריית drag/resize: **react-rnd** (בוגרת, קלה, תומכת מלא ב-RTL וב-snap).
- מבנה `spec`: `{ grid: {cols:12, rows:8}, background: "...", blocks: [{ id, type, x, y, w, h, z, props }] }`.
- רינדור: קומפוננטת `<CustomLayoutRenderer spec={...} />` שממפה `type` לרכיב הנכון (משתמשים ברכיבים המשותפים הקיימים — Canvas, Toolbar, EmojiRail, וכו').
- Undo/redo: היסטוריית `spec` ב-`useReducer` (עד 50 צעדים).
- תמונה קטנה: `html-to-image` על הקנבס לפני שמירה.
- הענן יופעל בשלב הראשון של הבנייה (Lovable Cloud + Auth בסיסי + טבלה + RLS).

## תוכנית ביצוע

1. הפעלת Lovable Cloud + טבלת `custom_layouts` + RLS + מסך התחברות פשוט.
2. הוצאת הרכיבים המשותפים (Canvas, Toolbar, EmojiRail, ReactionsFloat, useTimer) לקבצים נפרדים ב-`src/components/studio/`.
3. `CustomLayoutRenderer` — מרנדר פריסה מתוך JSON.
4. `LayoutEditor` — עורך ויזואלי מלא (רשת, drag/resize, פלטה, פאנל מאפיינים, undo/redo, שמירה, thumbnail).
5. עדכון `LayoutSwitcher` — טעינת פריסות מותאמות מהענן + כפתור עריכה + כפתור "פריסה חדשה".
6. אינטגרציה בעמוד הראשי: אם `layout` הוא ID של פריסה מותאמת → `CustomLayoutRenderer`, אחרת אחת מהשלוש המובנות.

## הערכת מורכבות

בינונית-גבוהה — כ-6-8 קבצים חדשים, ~700-900 שורות. הפעלת ענן + טבלה אחת + הרשאות. אין תלויות חיצוניות מסובכות (react-rnd + html-to-image בלבד).
